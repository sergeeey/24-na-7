# ADR 0001: Использование FastAPI для API

**Статус:** Принято  
**Дата:** 2024-02-17  
**Авторы:** Команда разработки

## Context

Требовался выбор фреймворка для REST API и WebSocket endpoints для проекта Reflexio 24/7.

## Decision

Выбран FastAPI как основной фреймворк для API.

**Причины:**
- Автоматическая генерация OpenAPI документации
- Встроенная поддержка WebSocket
- Высокая производительность (на основе Starlette и Pydantic)
- Простота использования и хорошая типизация
- Активная экосистема и сообщество

## Consequences

**Положительные:**
- Автоматическая документация API через Swagger UI
- Простота добавления новых endpoints
- Хорошая производительность для async операций
- Интеграция с Pydantic для валидации

**Отрицательные:**
- Зависимость от Python 3.11+ (но это соответствует требованиям проекта)
- Меньше готовых решений по сравнению с Django REST Framework
