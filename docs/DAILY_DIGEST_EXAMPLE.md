# Пример вечерней аналитики за день

Так выглядит ответ API `GET /digest/daily?date=YYYY-MM-DD` и экран «Итог дня» в приложении, когда за день есть несколько записей и по ним есть анализы (или fallback по транскрипциям).

---

## 1. Пример JSON (ответ API)

```json
{
  "date": "2026-02-06",
  "summary_text": "Продуктивный день: утром обсудили проект с командой, днём встреча с Игорем по дизайну — всё согласовали. К вечеру чувствовалась усталость, но доволен результатом.",
  "key_themes": ["проект", "встреча", "дизайн", "команда", "сроки"],
  "emotions": ["уверенность", "удовлетворённость", "усталость"],
  "actions": [
    { "text": "встреча с Игорем по макетам", "done": false },
    { "text": "отправить правки Кате до пятницы", "done": false },
    { "text": "подготовить слайды к понедельнику", "done": false }
  ],
  "total_recordings": 8,
  "total_duration": "14m 20s",
  "repetitions": []
}
```

---

## 2. Как это выглядит на экране «Итог дня»

**Итог дня**  
[Назад]

---

**Итог**

Продуктивный день: утром обсудили проект с командой, днём встреча с Игорем по дизайну — всё согласовали. К вечеру чувствовалась усталость, но доволен результатом.

---

**Ключевые темы**

- проект  
- встреча  
- дизайн  
- команда  
- сроки  

---

**Эмоции**

уверенность, удовлетворённость, усталость  

---

**Действия**

□ встреча с Игорем по макетам  
□ отправить правки Кате до пятницы  
□ подготовить слайды к понедельнику  

---

**Статистика**

Записей: 8, длительность: 14m 20s  

---

(Если в ответе есть `repetitions`, они выводятся блоком «Повторяется».)

---

## 3. Откуда берутся данные

| Поле | Источник |
|------|----------|
| `summary_text` | Либо склейка полей `summary` из таблицы `recording_analyses` по записям за день, либо (если анализов нет) общий текст транскрипций или краткий fallback. |
| `key_themes` | Уникальные `topics` из анализов записей или извлечённые факты. |
| `emotions` | Уникальные `emotions` из анализов или из фактов типа «эмоция». |
| `actions` | Список `actions` из анализов (поле `done` пока не заполняется автоматически — можно доработать). |
| `total_recordings` | Число транскрипций за выбранную дату. |
| `total_duration` | Сумма длительностей записей за день (формат «Xm Ys»). |
| `repetitions` | Пока пустой массив; в Phase 3 — повторяющиеся темы/паттерны по дням. |

Когда пользователь открывает «Итог дня» вечером, приложение запрашивает `GET /digest/daily?date=сегодня` и рисует экран по этому формату.
